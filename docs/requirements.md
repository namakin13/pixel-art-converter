# ドット絵コンバーター 要件定義書

最終更新: 2026-06-25

## 1. 概要

イラスト・写真などの画像を **ドット絵（ピクセルアート）に変換** し、さらに **その場で編集**・**ゼロから作成**・**アイコンとして書き出し** できる Windows デスクトップアプリ。Tauri 2 + React 19 + TypeScript で実装し、インストーラーまで配布する。

### 利用シーン
- 手持ちのイラストをドット絵風に変換したい
- 変換結果を微調整したい（はみ出し・色ムラの修正）
- ドット絵を一から描きたい
- 作ったドット絵をアプリ/サイト用のアイコン（.ico / PNG）にしたい

---

## 2. 機能要件

### 2.1 画像の入力
- **F-01** ファイル選択ダイアログから画像を読み込む（PNG / JPEG / WebP / BMP / GIF静止）
- **F-02** ウィンドウへのドラッグ＆ドロップで読み込む
- **F-03** クリップボードからの貼り付け（Ctrl+V）※追加提案
- **F-04** 入力画像は最大 HD 相当（1280×720）まで対応。超過分は読み込み時に自動縮小し、元アスペクト比を保持

### 2.2 ドット絵変換
- **F-10** 出力ドット解像度の指定（横ドット数。例: 16 / 32 / 48 / 64 / 96 / 128 / 256、および任意指定）
- **F-11** ピクセル化アルゴリズム: ブロック平均化（ダウンサンプル）＋最近傍拡大表示
- **F-12** 減色（カラー量子化）。手法は **メディアンカット** を既定とし、後述のパレットも選択可
- **F-13** ディザリング ON/OFF（Floyd–Steinberg など）※追加提案
- **F-14** 変換前後のプレビュー比較（スライダー / 並べ表示）
- **F-15** 明度・コントラスト・彩度の事前補正スライダー ※追加提案
- **F-16** アウトライン強調（輪郭抽出して縁取り）※追加提案

### 2.3 カラーパレット
- **F-20** パレット色数のプリセット: 2 / 4 / 8 / 16 / 32 / 64 / 128 / 256 色
- **F-21** 有名パレットプリセット同梱: PICO-8(16), NES, Game Boy(4), CGA, PixelArt 32 等
- **F-22** 自動生成パレット（画像から最頻色を抽出）
- **F-23** ユーザー定義パレットの作成・保存・読み込み（.hex / .gpl インポート対応）※追加提案
- **F-24** カラーロック（特定色を変換で保持）※追加提案

### 2.4 ピクセルエディタ（生成後編集 / ゼロから作成）
- **F-30** 変換結果をそのまま編集モードへ送る（生成後編集）
- **F-31** 空キャンバスから新規作成（サイズ・背景透過を指定）
- **F-32** ツール: ペン / 消しゴム / 塗りつぶし(バケツ) / スポイト / 矩形・直線 / 範囲選択・移動
- **F-33** カラーパレットパネル + 任意色ピッカー
- **F-34** レイヤー（最低: 単一レイヤー、将来: 複数レイヤー）※複数は追加提案
- **F-35** Undo / Redo（履歴）
- **F-36** グリッド表示 ON/OFF、ズーム（拡大縮小）、パン
- **F-37** 透過背景のチェッカーボード表示
- **F-38** 左右反転・上下反転・回転 ※追加提案
- **F-39** ミラー描画（対称ペン）※追加提案

### 2.5 アイコン書き出し
- **F-40** PNG 書き出し（等倍 / 整数倍スケール 2x/4x/8x 選択）
- **F-41** .ico マルチサイズ書き出し（16/32/48/64/128/256 を1ファイルに同梱）
- **F-42** よく使うアイコンサイズへの一括正方形リサイズ
- **F-43** 透過の維持
- **F-44** Windows アプリ用 / ファビコン用などの書き出しプリセット ※追加提案

### 2.6 プロジェクト保存
- **F-50** 作業状態（元画像・設定・編集レイヤー）を独自プロジェクト形式で保存/再開 ※追加提案
- **F-51** 直近に開いたファイルの履歴 ※追加提案

### 2.7 配布
- **F-60** Tauri bundle で Windows インストーラーを生成（NSIS `.exe` を既定、必要に応じ MSI も）
- **F-61** アプリアイコン・製品名・バージョン情報の設定

---

## 3. 非機能要件
- **N-01** 対応OS: Windows 10/11（64bit）
- **N-02** UI言語: 日本語（将来i18n可能な構造）
- **N-03** 完全オフライン動作（ネットワーク不要、画像はローカル処理のみ）
- **N-04** HD画像＋256色変換が数秒以内に完了する応答性
- **N-05** ダークUIを基本としたシンプルで触りやすい操作性

---

## 4. 画面構成（案）
1. **ホーム** — 「画像を変換」「新規作成」「最近のプロジェクト」
2. **変換画面** — 左: 入力/プレビュー、右: 解像度・パレット・ディザ等の設定パネル
3. **エディタ画面** — 中央: キャンバス、左: ツール、右: パレット/レイヤー
4. **書き出しダイアログ** — PNG / .ico / サイズ・倍率選択

---

## 5. 技術方針
- **画像処理**: まずフロント（Canvas API / TypeScript）で実装。パフォーマンス上必要なら Rust 側（`image`, `imageproc` クレート）へ移管
- **状態管理**: React の状態 + 必要に応じ軽量ストア（Zustand 等）
- **ファイルI/O・ダイアログ**: Tauri プラグイン（dialog / fs）
- **.ico 生成**: Rust 側（`ico` クレート）またはフロントで複数PNGを束ねる
- **D&D**: Tauri のファイルドロップイベント

---

## 6. 開発フェーズ（ロードマップ）

| Phase | 内容 | 主な完了条件 |
|-------|------|--------------|
| **P1** | 基盤＋変換 | 画像読込(ファイル/D&D)→ピクセル化→減色→プレビュー→PNG書き出し |
| **P2** | パレット強化 | 色数プリセット・有名パレット・自動抽出・ディザリング |
| **P3** | エディタ | ペン/消し/バケツ/スポイト, Undo/Redo, グリッド/ズーム, 透過 |
| **P4** | 新規作成＋アイコン | 空キャンバス新規作成, .ico マルチサイズ書き出し |
| **P5** | 配布 | アプリアイコン設定, NSIS インストーラー生成, バージョニング |
| **P6** | 追加機能 | レイヤー, プロジェクト保存, 補正/反転/ミラー 等 |

---

## 7. 追加提案機能まとめ（ユーザー要望「便利な機能の提案」への回答）
- クリップボード貼り付け / 直近ファイル履歴
- ディザリング、明度・コントラスト・彩度補正、アウトライン強調
- ユーザー定義パレットの保存・インポート、カラーロック
- 反転・回転・対称(ミラー)描画、複数レイヤー
- プロジェクト形式での作業保存
- 書き出しプリセット（Windowsアプリ用 / ファビコン用）

> これらは P1〜P5 のコア完成後に優先度を相談しつつ取り込む想定。
